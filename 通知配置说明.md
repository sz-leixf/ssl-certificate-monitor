## 企业微信配置 API

### 1. 获取企业微信配置

**接口**: `GET /api/notification/config/wechat`

**描述**: 获取当前的企业微信配置

**响应示例**:
```json
{
  "success": true,
  "data": {
    "webhookUrl": "已配置",
    "enabled": true,
    "mention": {
      "mobiles": [],
      "userids": []
    }
  }
}
```

### 2. 更新企业微信配置

**接口**: `PUT /api/notification/config/wechat`

**请求头**: `Content-Type: application/json`

**请求体**:
```json
{
  "webhookUrl": "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY",
  "enabled": true,
  "mention": {
    "mobiles": ["13800000000", "13900000000"],
    "userids": ["user001", "user002"]
  }
}
```

**参数说明**:
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-------|------|
| webhookUrl | string | 否 | 企业微信机器人 Webhook URL |
| enabled | boolean | 否 | 是否启用通知，默认 true |
| mention | object | 否 | @成员配置 |
| mention.mobiles | array | 否 | @成员的手机号数组 |
| mention.userids | array | 否 | @成员的用户ID数组 |

**成功响应**:
```json
{
  "success": true,
  "data": { ... },
  "message": "企业微信配置已更新"
}
```

## 使用示例

### PowerShell 示例

#### 1. 获取企业微信配置

```powershell
$response = Invoke-WebRequest -Uri "http://localhost:3001/api/notification/config/wechat" -UseBasicParsing
$config = $response.Content | ConvertFrom-Json
$config.data
```

#### 2. 更新企业微信配置

```powershell
$body = @{
    webhookUrl = "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY"
    enabled = $true
    mention = @{
        mobiles = @("13800000000", "13900000000")
        userids = @("user001", "user002")
    }
} | ConvertTo-Json -Depth 3

$response = Invoke-WebRequest -Uri "http://localhost:3001/api/notification/config/wechat" `
    -Method PUT `
    -Body $body `
    -ContentType "application/json" `
    -UseBasicParsing

$result = $response.Content | ConvertFrom-Json
if ($result.success) {
    Write-Host "配置更新成功"
} else {
    Write-Host "更新失败: $($result.error)"
}
```

#### 3. 发送测试通知

```powershell
$body = @{
    webhook = "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY"
} | ConvertTo-Json

$response = Invoke-WebRequest -Uri "http://localhost:3001/api/test-notification" `
    -Method POST `
    -Body $body `
    -ContentType "application/json" `
    -UseBasicParsing

$result = $response.Content | ConvertFrom-Json
if ($result.success) {
    Write-Host "测试通知发送成功"
} else {
    Write-Host "发送失败: $($result.error)"
}
```

### JavaScript 示例

```javascript
// 获取企业微信配置
async function getWechatConfig() {
  const response = await fetch('http://localhost:3001/api/notification/config/wechat');
  const result = await response.json();
  return result.data;
}

// 更新企业微信配置
async function updateWechatConfig(webhookUrl, enabled, mention) {
  const response = await fetch('http://localhost:3001/api/notification/config/wechat', {
    method: 'PUT',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      webhookUrl: webhookUrl,
      enabled: enabled,
      mention: mention
    })
  });

  const result = await response.json();
  if (result.success) {
    console.log('配置更新成功');
  } else {
    console.error('更新失败:', result.error);
  }
}

// 发送测试通知
async function sendTestNotification(webhookUrl) {
  const response = await fetch('http://localhost:3001/api/test-notification', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      webhook: webhookUrl
    })
  });

  const result = await response.json();
  if (result.success) {
    console.log('测试通知发送成功');
  } else {
    console.error('发送失败:', result.error);
  }
}
```

## 消息模板变量

### 可用变量列表

所有消息模板支持以下变量：

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `{domain_name}` | 域名 | example.com |
| `{valid_until}` | 到期时间 | 2026-06-30 23:59:59 |
| `{days_until_expiry}` | 剩余天数 | 30 |
| `{issuer}` | 证书颁发者 | DigiCert |
| `{subject}` | 证书主题 | CN=example.com |
| `{send_time}` | 发送时间 | 2026-03-25 10:00:00 |
| `{error}` | 错误信息 | Connection timeout |

### 变量说明

- `{domain_name}`: 监控的域名名称
- `{valid_until}`: 证书到期时间（格式化为中文时间）
- `{days_until_expiry}`: 距离到期的天数
- `{issuer}`: 证书颁发机构
- `{subject}`: 证书主题信息
- `{send_time}`: 消息发送时间（所有通知类型统一使用此变量）
- `{error}`: 证书检查失败时的错误信息

## 故障排查

### 问题：通知未发送

**排查步骤**：
1. 检查 Webhook URL 是否完整且正确
2. 检查 `enabled` 是否为 `true`
3. 检查企业微信机器人是否还在群中
4. 查看后端日志中的错误信息

**解决方案**：
- 在企业微信群中重新添加机器人
- 确认 Webhook URL 包含完整地址和密钥
- 检查网络连接

### 问题：@成员不生效

**排查步骤**：
1. 确认成员是否在群中
2. 检查手机号或用户ID格式是否正确
3. 确认是企业内部群

**解决方案**：
- 确保成员是企业微信用户
- 手机号格式：11位数字，不带区号
- 用户ID格式：企业微信内部用户ID

### 问题：Webhook URL 失效

**排查步骤**：
1. 检查机器人是否被删除
2. 检查机器人权限是否被限制
3. 测试 URL 是否可以直接访问

**解决方案**：
- 在企业微信群中重新添加机器人
- 联系企业管理员检查机器人配置
- 定期更换 Webhook 密钥

### 问题：频繁发送通知

**排查步骤**：
1. 检查提醒阈值设置是否过低
2. 检查是否配置了多个域名
3. 查看通知发送频率

**解决方案**：
- 合理设置提醒阈值（建议 10-30 天）
- 批量检查时注意通知频率
- 使用定时检查减少手动触发

## 🔒 安全建议

1. **Webhook URL 保护**
   - 不要在公开代码仓库中提交 Webhook URL
   - 使用环境变量存储敏感信息
   - 定期更换 Webhook 密钥

2. **通知限制**
   - 企业微信机器人每分钟最多 20 条消息
   - 合理设置提醒阈值，避免频繁通知

3. **数据备份**
   - 定期备份 `data/notification-config.json` 文件
   - 保存多个版本的配置备份

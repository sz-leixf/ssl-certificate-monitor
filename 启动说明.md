## 📱 配置企业微信

### 创建企业微信机器人

1. 在企业微信群中，点击右上角 "..."
2. 选择"添加群机器人"
3. 设置机器人名称和头像
4. 创建后获取 Webhook URL

### 配置通知

1. 在系统左侧菜单点击"通知设置"
2. 将 Webhook URL 填入到"企业微信 Webhook URL"输入框
3. （可选）填写 @成员的手机号或用户ID
4. 点击"测试通知"验证配置
5. 确认企业微信群收到测试消息
6. 点击"保存设置"

### @成员配置

系统支持在通知时 @ 指定成员：

- **手机号**: 13800000000,13900000000
- **用户ID**: user001,user002
- **格式**: 多个成员用逗号分隔

## ❓ 常见问题

### 端口被占用

如果遇到端口被占用，使用以下命令查找并关闭进程：

```powershell
# Windows - 查看 3001 端口占用
netstat -ano | findstr :3001

# Windows - 关闭进程（替换 PID 为实际进程 ID）
taskkill /F /PID <进程ID>
```

```bash
# Linux - 查看 3001 端口占用
netstat -tunlp | grep 3001

# Linux - 关闭进程
kill -9 <PID>
```

### 修改默认端口

编辑 `server/index.js` 文件：

```javascript
const PORT = process.env.PORT || 3002; // 改为其他端口
```

### 调整证书检查超时时间

编辑 `server/utils/certificateChecker.js` 文件：

```javascript
req.setTimeout(30000); // 默认 20 秒，可调整为其他值
```

### 修改定时检查间隔

在 Web 界面的"定时设置"页面中修改：
- **检查间隔**: 1-30 天
- **检查时间**: 0-23 时，0-59 分

### 数据库文件损坏

```bash
# 1. 停止服务
# Ctrl+C

# 2. 备份现有数据
cd data
cp domains.db domains.db.backup

# 3. 删除损坏的数据库
rm domains.db

# 4. 重启服务
npm start

# 系统会自动创建新的数据库文件
```

## 💾 数据存储

### 数据文件位置

所有数据存储在 `data/` 目录下：

```
data/
├── domains.db                      # 域名列表数据库
├── certificate_checks.db           # 证书检查记录
├── notifications.db                # 通知发送记录
├── notification-config.json        # 通知配置（企业微信、消息模板）
└── auth-config.json               # 认证配置（用户名、密码、登录状态）
```

### 数据备份

```bash
# 备份所有数据
tar -czf ssl-monitor-backup-$(date +%Y%m%d).tar.gz data/

# 恢复数据
tar -xzf ssl-monitor-backup-20260402.tar.gz
```

### 定期备份建议

建议每周备份一次数据：

```bash
# Windows 任务计划程序
# 创建任务每周日凌晨 2 点执行备份

# Linux crontab
# 编辑 crontab
crontab -e

# 添加以下行（每周日凌晨 2 点备份）
0 2 * * 0 tar -czf /backup/ssl-monitor-$(date +\%Y\%m\%d).tar.gz /path/to/data/
```

### 生产环境部署

#### 使用 PM2（进程管理器）

```bash

pm2 start server/index.js --name ssl-monitor

# 查看日志
pm2 logs ssl-monitor

# 重启服务
pm2 restart ssl-monitor

# 停止服务
pm2 stop ssl-monitor
```

#### 使用 Docker

详细 Docker 部署说明请参考 [docker部署说明.md](./docker部署说明.md)

#### 使用 Nginx 反向代理

配置 Nginx 反向代理，支持 HTTPS：

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    # SSL 证书配置
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    # SSL 优化配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    
    # 反向代理
    location / {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

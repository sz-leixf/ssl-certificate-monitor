# 企业微信配置功能总结

## ✅ 已完成功能

### 1. 配置管理模块 (`server/utils/configManager.js`)

**核心功能：**
- ✅ 配置文件的自动创建和管理
- ✅ 企业微信配置的增删改查
- ✅ 消息模板的增删改查
- ✅ 配置验证功能
- ✅ 配置重置为默认值

**配置文件位置：**
```
data/notification-config.json
```

### 2. 企业微信配置管理

**配置项：**
- `webhookUrl` - 企业微信机器人 Webhook URL
- `enabled` - 通知开关（true/false）
- `mention.mobiles` - @成员的手机号数组
- `mention.userids` - @成员的用户ID数组

**API 端点：**
- `GET /api/notification/config/wechat` - 获取配置
- `PUT /api/notification/config/wechat` - 更新配置

### 3. 消息模板管理

**支持的模板类型：**
1. `alert` - 证书到期提醒（默认）
2. `expired` - 证书已过期
3. `warning` - 证书即将到期（7天内）
4. `success` - 证书检查成功
5. `error` - 证书检查失败
6. `test` - 测试通知

**模板功能：**
- ✅ Markdown 格式支持
- ✅ 模板变量替换
- ✅ 自定义模板内容

**API 端点：**
- `GET /api/notification/config/templates` - 获取所有模板
- `GET /api/notification/config/templates/:type` - 获取指定模板
- `PUT /api/notification/config/templates/:type` - 更新模板
- `POST /api/notification/config/templates/:type/test` - 测试模板

### 4. 模板变量支持

**可用变量：**
| 变量名 | 说明 | 示例 |
|--------|------|------|
| `{domain_name}` | 域名 | example.com |
| `{valid_until}` | 到期时间 | 2026-06-30 23:59:59 |
| `{days_until_expiry}` | 剩余天数 | 30 |
| `{issuer}` | 证书颁发者 | DigiCert |
| `{subject}` | 证书主题 | CN=example.com |
| `{send_time}` | 发送时间 | 2026-03-25 10:00:00 |
| `{error}` | 错误信息 | Connection timeout |

> **注意**: 所有时间相关变量已统一为 `{send_time}`

### 5. 集成到通知系统

**更新的模块：**
- `server/utils/wechatBot.js` - 企业微信通知模块
  - 使用配置管理模块获取配置
  - 根据证书状态自动选择模板
  - 支持模板变量替换

**通知流程：**
1. 检测到证书过期或即将到期
2. 根据状态选择合适的模板（alert/expired/warning）
3. 替换模板变量
4. 发送企业微信通知
5. 记录通知历史

### 6. 新增 API 接口

**配置管理 API：**
```
GET    /api/notification/config              - 获取所有配置
GET    /api/notification/config/wechat        - 获取企业微信配置
PUT    /api/notification/config/wechat        - 更新企业微信配置
GET    /api/notification/config/templates     - 获取所有模板
GET    /api/notification/config/templates/:type - 获取指定模板
PUT    /api/notification/config/templates/:type - 更新模板
POST   /api/notification/config/validate     - 验证配置
POST   /api/notification/config/reset        - 重置为默认值
```

## 🚀 使用示例

### 1. 配置企业微信

```powershell
# 更新企业微信配置
$body = @{
    webhookUrl = "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY"
    enabled = $true
    mention = @{
        mobiles = @("13800000000")
        userids = @("user001")
    }
} | ConvertTo-Json -Depth 3

Invoke-WebRequest -Uri "http://localhost:3001/api/notification/config/wechat" `
    -Method PUT `
    -Body $body `
    -ContentType "application/json" `
    -UseBasicParsing
```

### 2. 自定义消息模板

```powershell
# 更新警告模板
$body = @{
    title = "🚨 紧急：证书即将过期！"
    content = "### 🚨 紧急通知

> **域名**: {domain_name}
> **剩余天数**: {days_until_expiry} 天
> **到期时间**: {valid_until}
> **发送时间**: {send_time}

请立即处理！"
    msgtype = "markdown"
} | ConvertTo-Json

Invoke-WebRequest -Uri "http://localhost:3001/api/notification/config/templates/warning" `
    -Method PUT `
    -Body $body `
    -ContentType "application/json" `
    -UseBasicParsing
```

### 3. 测试消息模板

```powershell
# 测试指定模板
Invoke-WebRequest -Uri "http://localhost:3001/api/notification/config/templates/test/test" `
    -Method POST `
    -UseBasicParsing
```

## ✨ 功能特点

1. **无需重启服务** - 配置修改后立即生效
2. **持久化存储** - 配置保存在 JSON 文件中
3. **灵活的模板** - 支持自定义消息内容和格式
4. **安全的配置** - Webhook URL 在获取接口中会被隐藏
5. **实时测试** - 可以测试任何模板而无需实际触发通知
6. **完善的文档** - 提供详细的使用说明和示例

## 🔧 技术实现

### 配置管理
- 使用 JSON 文件存储配置
- 自动合并默认配置
- 支持配置验证

### 模板引擎
- 简单的字符串替换机制
- 支持 Markdown 格式
- 变量占位符 `{variable_name}`

### API 设计
- RESTful 风格
- 统一的响应格式 `{ success: boolean, data: any, message: string }`
- 完善的错误处理

## 📝 注意事项

1. **配置文件权限** - 确保 `data/` 目录有写入权限
2. **Webhook URL 安全** - 密钥信息请妥善保管
3. **模板变量** - 自定义模板时请使用正确的变量名
4. **消息格式** - 确保 Markdown 格式正确
5. **通知开关** - 可以通过 `enabled` 字段临时禁用通知
